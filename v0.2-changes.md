# Mosaic spec v0.2 ‚Äî draft changes

This document summarises the additive, non-breaking sections introduced in the `v0.2-draft` branch. None of v0.1's normative content has been altered.

## Summary

Four additions, each marked `(v0.2 DRAFT)` in `spec.md` for reviewer visibility.

### 1. `LayoutSpec` body ‚Äî ¬ß4.6a + ¬ß5.1a

v0.1 declared `"layouts": { "<name>": <LayoutSpec>, ... }?` in `mosaic.json` but left `LayoutSpec` undefined. v0.2 fleshes it out as a responsive CSS-Grid-shaped template: a `grid` object (columns + gap token), an `areas` map keyed by breakpoint (using `grid-template-areas` row syntax), an optional `breakpoints` map (key ‚Üí min viewport width in pixels), and `sectionDefaults.area` for unassigned sections. Section instances opt into a named area via the new optional `sectionLayout: { area, span? }` field on `SectionInstance` (¬ß5.1a). Implementable in browser-native CSS Grid; non-CSS renderers MAY approximate as a vertical stack. Level 1 readers MAY ignore `layouts` entirely.

### 2. `tokenOverrides` cascade ‚Äî ¬ß4.2a + ¬ß5.2 + ¬ß5.1 SectionInstance field

Tokens were site-level only in v0.1. v0.2 adds a scope cascade: site ‚Üí page ‚Üí section. (Slot-level was considered and deferred ‚Äî see open question below.) `tokenOverrides` is a two-level map identical in shape to `mosaic.json#tokens` and lives optionally on page records (¬ß5.2) and on `SectionInstance` (¬ß5.1). Resolution is last-write-wins along the cascade. Overrides MUST only override token NAMES that already exist at the site level ‚Äî introducing a new name via override is invalid (readers ignore + warn). Renderers SHOULD emit scoped CSS custom property declarations on the page/section wrapper. Token VALUE changes do NOT bump `schemaVersion`. Support is OPTIONAL for Level 1.

### 3. `freeform` canonical block type ‚Äî ¬ß10b

A canonical block type for absolute-positioned design-heavy content (covers, editorial inserts, posters, art-directed callouts). Declared aspect-ratio canvas, list of `freeformItem` structs (`kind` ‚àà `"text" | "asset" | "shape"`, plus `position: { x, y, w, h, z?, rotation?, units? }`). Default coordinate basis is percent of canvas; `units: "px"` opts into absolute pixels. Renderers SHOULD use `position: absolute` against a `position: relative` canvas. Spec explicitly documents the portability tradeoff: `freeform` is the LEAST portable canonical block across surfaces ‚Äî headless consumers (¬ß6.1) SHOULD include it in their skip set unless they specifically support it. Level 1 readers MAY treat it as unknown.

### 4. Consumer capability declaration ‚Äî silent skip ‚Äî ¬ß6.1

v0.1 ¬ß6 says invalid sections surface via a warnings channel, and unknown block types render as `<missing template>`. That's correct for engine-default renderers but wrong for headless consumers (custom renderers, mobile, AI, RSS, search indexers) that are partial projections of a document. v0.2 adds ¬ß6.1: a consumer MAY declare a supported block types set and MUST silently drop unsupported blocks from the render tree (no fallback, no placeholder, no warning for the unsupported `blockType` itself ‚Äî separate from ¬ß6 validation, which still applies to supported sections). Canonical pattern: a `SKIP_BLOCKS` set on the consumer side. Pass-through serialisers (Level 2 writers) SHOULD preserve unsupported sections so other consumers still see them ‚Äî silent skip applies to rendering, not storage. `mosaic.json` MUST NOT declare per-consumer skip rules. This contract is OPTIONAL for engine-default renderers.

## Open questions

- **Slot-level `tokenOverrides`** ‚Äî the cascade in ¬ß4.2a stops at section. Slot-level was considered (e.g. "make this one button's `color.accent` red") but rejected for now: it bloats the slot value schema (every slot would need an optional override sidecar), it conflicts with translatable slot serialisation, and the same effect is already achievable by overriding at the section that holds the slot. Keep section as the narrowest scope until a concrete authoring need emerges.
- **`freeform` units mixing** ‚Äî current spec lets each `freeformItem` declare its own `units` independently. Likely fine, but the visual outcome of mixing percent and px items in the same canvas is undefined under arbitrary aspect rescaling. Open question whether `units` should be a canvas-level field instead of per-item.
- **`LayoutSpec.areas` per-breakpoint row-count mismatch** ‚Äî spec says row counts MAY differ across breakpoints; renderers will then have rows that "appear" at large widths only. Reasonable for CSS Grid but raises questions for non-CSS approximators about how to order them in the fallback stack.
- **`sectionLayout.span` semantics** ‚Äî declared as a column span integer, but with named areas (which already control geometry) `span` is mostly redundant. Kept as an optional refinement, but may be removable if no real authoring need surfaces.
- **`freeform` and headless skip default** ‚Äî ¬ß10b.6 says headless consumers SHOULD include `freeform` in their skip set. Worth considering whether the spec should go further and RECOMMEND it as the default behavior in the ¬ß6.1 canonical pattern (i.e. ship `freeform` in the default skip set for headless consumer reference implementations).

## Peer-review fixes folded into v0.2-draft (2026-05-11)

After an external peer review (see `REVIEW.md`), the following spec-correctness fixes landed in v0.2-draft. These are NOT new features ‚Äî they close gaps in v0.1 that the review surfaced.

- **Section numbering** ‚Äî duplicate `## 10` resolved. Versioning is now ¬ß11; canonical hashing inserted as new ¬ß12; reference implementation moved to ¬ß13; open questions moved to ¬ß14. ¬ß10a / ¬ß10b moved physically above ¬ß11 Versioning so document order matches numbering.
- **Slug rule (C5)** ‚Äî ¬ß2 narrowed: page slugs MUST start with `/`; collection record ids MUST be bare. README blog example is now consistent.
- **"Block algorithm" (C3)** ‚Äî undefined term removed from Level 1 MUST in ¬ß1. Now reads "Validate `mosaic.json` against ¬ß4 and each content record against ¬ß5."
- **"Tier 2 storage" / "hybrid storage" (C4)** ‚Äî undefined terminology removed from ¬ß7.3. Replaced with concrete language: content-addressed block storage is engine-OPTIONAL; refs without backing storage resolve to `{ kind: "block", content: null }`.
- **Reference string grammar (C2)** ‚Äî new normative Appendix C. ABNF for `asset:`, `ref:`, `block:` shapes. Normalisation rules (trim, strip leading `./`, reject `..`, reject empty segments). Case-sensitivity rule. Reviewer flagged this as the highest-leverage gap.
- **Canonical hashing (new ¬ß12)** ‚Äî RFC 8785 JCS for canonical JSON encoding. Defines `publishedHash` computation (over `slots` only, unknown fields included). Defines asset manifest sha256 (raw bytes). Defines `block:sha256-<hash>` derivation. Closes the underspec gap that made hash-based comparison non-portable.
- **Misc nits** ‚Äî title bumped to "0.2 (draft ‚Äî normative; subject to change pre-v1.0)"; Windows path-separator rule added to ¬ß2; ¬ß10.4 cross-ref corrected (¬ß6 ‚Üí ¬ß6.1); Appendix B gained a v0.2 entry.

## Still-open issues from peer review (deferred to v1.0)

The peer review flagged additional issues that this draft does NOT address. They are intentionally deferred so v0.2 stays additive and ships:

- **C6** ‚Äî empty `blockTypes` + empty `pages` interaction (whether a zero-content site is valid).
- **C7** ‚Äî asset manifest absence vs SHOULD-tolerate clause (¬ß3 vs ¬ß8 collision).
- **M1** ‚Äî `richtext` value-level discriminator (markdown vs portable-text without SlotDef).
- **M2** ‚Äî `i18n.fallback` semantics (per-slot, per-record, per-route).
- **M3** ‚Äî `globalsOverride` example shows duplicate JSON keys; needs typed-union presentation.
- **M4** ‚Äî globals ordering when multiple share a position; `above`/`below` not always declared.
- **M5** ‚Äî `block:` ref `content` shape across README vs ¬ß7.3.
- **M6** ‚Äî `freeform` `units` mixing under canvas rescale (already in this doc's open questions below).
- **M7** ‚Äî `tokenOverrides` partial-shadow case (group has known + unknown names).
- **M8** ‚Äî `state: draft` section interaction with `sectionLayout`, `publishedHash`, `before-section:<n>`.
- **Validation order** ‚Äî when a section is invalid AND its block type is in a ¬ß6.1 skip set, what surfaces?
- **JSON Schema for `mosaic.json`** ‚Äî formal validator-via-JSON-Schema, planned in ¬ß14 (was ¬ß12); not delivered in v0.2.
- **Locale fallback at the slot level** ‚Äî see M2.
- **Schema migration** ‚Äî what happens when a renderer reads a higher `schemaVersion` than it understands.
